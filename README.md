# 家庭消费需求模型 Skill 构建指南

> 本文档介绍如何基于专家知识构建家庭消费领域的可执行 Skill。

---

## 一、概述

家庭消费需求模型 Skill 是用于满足家庭场景下用户需求的标准化能力单元。每个 Skill 包含：

- **Intent 定义**：标准化用户意图
- **能力清单**：支撑意图实现的最小能力集合
- **执行逻辑**：能力筛选与组合规则

---

## 二、通用构建流程

### 阶段一：专家认证

确认专家身份，确保只有授权专家才能创建 Skill。

**输入**：专家ID + 专家姓名  
**校验**：在 `experts.json` 中验证专家身份

---

### 阶段二：需求物料解析

收集专家知识素材，包括：

- 文本描述（概念、策略、原则）
- 音频/视频/图片素材
- 专家经验分享

**输出**：
- 核心结论摘要
- 关键要点清单
- 知识库路径

---

### 阶段三：意图构建（Intent）

将非结构化需求转化为标准化 Intent。

#### 1. 意图基础信息
- **意图领域**：从 12 个领域选择（居家环境管理、家电设备管理、休闲娱乐、餐饮美食、本地生活、实物购物消费、亲子教育服务、家庭照护服务、医疗健康、家庭出行、家庭政务、家居物品运维）
- **意图名称**：「动词 + 名词」结构
- **意图描述**：「用户角色 + 场景 + 问题」

#### 2. 六维约束
| 维度 | 说明 |
|------|------|
| 功能 | 必须具备或禁止的功能点 |
| 数量 | 频次、数量、范围限制 |
| 质量 | 效果、准确率、成功率要求 |
| 交付 | 响应速度、输出形式、时效性 |
| 情感 | 用户体验、语气风格、情绪感受 |
| 预算 | 成本、资源消耗、预算限制 |

---

### 阶段四：能力生成

基于 Intent 拆解出支撑能力。

#### 1. 候选能力生成
从三个类别生成候选能力：
- **需求理解能力**：识别、分析用户需求
- **分析推理能力**：推理、计算、评估
- **解决方案能力**：生成、执行、推荐

#### 2. 能力标准化
- 命名规范：「对象 + 行为 + 能力」
- 禁止场景化能力（如"儿童早餐推荐"）

#### 3. 能力库检索
- 相似度 ≥ 0.8 → 必须复用
- 相似度 0.6~0.8 → 优先复用
- 相似度 < 0.6 → 允许新建

#### 4. 约束映射
能力约束必须继承自 Intent 约束，禁止新增独立约束。

---

### 阶段五：Skill 生成

整合 Intent + 能力清单，生成可执行 Skill。

#### Skill 结构
```markdown
## Skill：xxx

### 一、Skill说明
一句话用途说明

### 二、输入（Runtime Input）
- 用户原始意图
- 用户私域数据

### 三、内置知识
- 意图结构（完整复制）
- 能力清单（完整复制）

### 四、执行逻辑
- 约束解析
- 能力筛选
- 能力组合
- 结果校验

### 五、输出
- 最终需能力列表
- 覆盖情况说明
```

---

### 阶段六：保存与发布

1. 保存 Skill 到 `skills/{Skill名称}/SKILL.md`
2. 创建知识库目录 `knowledge-base/{意图名称}-{专家姓名}-{专家ID}/`
3. 新能力入库 `capability-library.json`
4. 推送到 Git 分支

## 三、构建示例：放纵餐家庭消费模型

下面展示一个完整的 Skill 构建案例。

---

### 3.1 专家认证

- **专家ID**：EXP002
- **专家姓名**：李营养师
- **专长**：营养配餐

---

### 3.2 需求物料解析

**收集的专家知识：**

| 类别 | 内容 |
|------|------|
| **概念** | 放纵餐（Cheat Meal）- 长期饮食控制中周期性安排的相对自由饮食 |
| **目的** | 心理调节 + 长期可持续性 + 代谢刺激（部分人群） |
| **原则** | 放纵≠失控 / 允许高热量但有度 / 需结构性规划 |
| **策略** | 进食顺序（蛋白→蔬菜→高热量）、份量70%-80%、每周1-2次、运动后/白天、前置高蛋白 |
| **误区** | 当放纵日、情绪驱动、极端补偿 |
| **边界** | 单次热量≤1.5-2倍、不连续两餐、避免高糖+高脂 |
| **风险人群** | 血糖异常、暴食倾向、医疗饮食人群 |

---

### 3.3 意图构建

**生成的 Intent：**

| 维度 | 内容 |
|------|------|
| **意图领域** | 餐饮美食 |
| **意图名称** | 规划放纵餐饮食方案 |
| **意图描述** | 为有长期饮食控制需求的家庭用户提供周期性放纵餐的结构化规划与建议，平衡心理调节需求与健康边界，避免失控和极端补偿 |
| **功能约束** | 支持进食顺序建议、份量控制提醒、频率规划、时间建议、前置策略指导 |
| **数量约束** | 每周1-2次，单次热量不超过日常1.5-2倍 |
| **质量约束** | 避免高糖+高脂组合，识别风险人群并给出警告 |
| **交付约束** | 输出结构化放纵餐计划（含菜品/时间/份量/注意事项） |
| **情感约束** | 缓解饮食压力，提供正向鼓励，避免焦虑 |
| **预算约束** | 可选，根据家庭消费能力适配 |

---

### 3.4 能力生成

**候选能力：**

| # | 能力名称 | 描述 |
|---|----------|------|
| 1 | 饮食需求识别能力 | 识别用户的放纵餐需求类型、场景、周期 |
| 2 | 放纵餐策略匹配能力 | 根据用户情况匹配进食顺序、份量、频率等策略 |
| 3 | 热量边界计算能力 | 计算单次放纵餐的热量上限（日常1.5-2倍） |
| 4 | 风险人群评估能力 | 识别血糖异常、暴食倾向等风险人群并预警 |
| 5 | 饮食方案生成能力 | 生成结构化放纵餐计划（菜品、时间、份量） |
| 6 | 心理健康引导能力 | 提供正向鼓励，缓解饮食焦虑 |

**能力库检索：**
- CAP_002 饮食需求识别能力 → 复用
- CAP_012 饮食方案生成能力 → 复用
- CAP_023~CAP_026 → 新建入库

---

### 3.5 Skill 生成

**输出 Skill：** `skills/放纵餐家庭消费模型/SKILL.md`

核心执行逻辑：
```
约束解析 → 能力筛选 → 能力组合 → 结果校验
```

---

### 3.6 保存与发布

|------|------|
| Skill | `skills/放纵餐家庭消费模型/SKILL.md` |
| 知识库 | `knowledge-base/规划放纵餐饮食方案-李营养师-EXP002/` |
| 能力库 | `capability-library.json` (新增 CAP_023~CAP_026) |
